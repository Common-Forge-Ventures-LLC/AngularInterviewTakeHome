Instructions

Your goal is to create an application where we can see a list of employees and edit their data. This list should allow for the selection of an employee, which would then show more details for him/her. With an employee selected, we should be able to edit his/her data. By clicking save we should see the changes reflected in the employee list. We should also have the ability to delete an employee and add an employee. Here is a link to the design: Design


Getting Started

  1. Please create a new branch off of the Develop branch. The new branch name should be /applicant/{{'{'}}yourFirstName{{'}'}}-{{'{'}}yourLastName{{'}'}}-{{'{'}}todaysDate{{'}'}}
  2. When you are done make sure to push your changes to the repository and make pull request into Develop.

Objectives

  1. Display selectable list of all employees. This list will show the employee names and positions.
  2. A selected employee should then show additional details in the card to the right.
  3. All employee Data should be able editable and saved.
  4. We should be able to create a new Employee, that also shows up in the left hand list once created.
  5. We should be able to remove an employee, that is also removed from the left hand list after deletion.

Bonus

  • Match the design as best you can.

Additional info and Rules

  • An employee route and blank component have already been created for you to help you get started.
  • The employee data can be found in the employee.service.ts file. (Please keep the data in this file.)
  • The list of available positions and states data can be found in the data.service.ts file. (Please keep the data in this file.)
  • Must use/create at least 1 reusable component.
  • The list of employees and the employee details must be in separate components.
  • The design uses Angular Material Design styles. Angular Material has already been installed. Feel free to use their styles if you choose.
  • The icons/images you need in order to match the design are in the /assets/images folder
  • There is no database so all changes will be lost on page refresh. This is ok.
